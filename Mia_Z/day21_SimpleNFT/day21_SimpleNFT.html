Day21 SimpleNFT（迷你 ERC-721）教学面板

与 day21_SimpleNFT.sol 配套：NFT可类比为「带全局编号的所有权登记簿」——每个 tokenId 对应唯一一条记录，ownerOf 查当前持有人；balanceOf 统计某地址名下有多少个编号。 与 C 语言类比：mapping(uint256 => address) 像以整数为键的稀疏表；transferFrom 像带权限检查的指针改挂（改「谁拥有这个 ID」）。

接口 IERC721类似 C 头文件里的函数表约定：钱包、交易所、浏览器都按同一套函数名与事件名交互。安全转账 safeTransferFrom 在接收方是合约时会回调 onERC721Received，避免把 NFT 打进「不会处理 NFT 的合约」里锁死；像嵌入式里给对端发消息前先看对端是否实现了某握手协议。

本教学合约的 mint 未加 onlyOwner，便于本地随便铸；主网项目务必加权限或销售逻辑。技术栈：原生 HTML + JS，ethers v6（CDN）。若直接打开文件遇 RPC CORS，可用 VS Code Live Server 或 npx serve。

2. 教学步骤（按顺序）

阶段 A — 部署

  1. 在 Remix / Hardhat 部署 SimpleNFT，构造参数填入 name、symbol（如 LearnNFT / LNFT）。
  2. 将合约地址填入本节上方；链 ID 按实际网络填写。
  3. 点「读取 name / symbol」确认连到正确合约。

阶段 B — 铸造与查询

  1. 连接 MetaMask；在「铸造」卡片填写接收地址与 tokenURI 字符串（可用 IPFS 链接或任意教学用 URL）。
  2. 发送 mint 后，页面会从交易收据里解析 Transfer 事件得到新 tokenId（合约内部计数器为 private，链外需靠事件或浏览器）。
  3. 在「按 tokenId 查询」输入编号，查 ownerOf、tokenURI、当前地址的 balanceOf。

阶段 C — 授权与转移（可选）

  1. approve：授权某一地址转移单个 tokenId。
  2. setApprovalForAll：授权某操作员转移你名下全部 NFT（市场常用）。
  3. transferFrom / safeTransferFrom：在已授权或本人为 owner 时转移；对合约地址优先用 safe 路径做演示。

7. 测试验收清单（自学对照）

部署：构造参数正确，浏览器可读到 name、symbol。
铸造：成功则收据中有 Transfer(0x0, to, tokenId)；ownerOf(tokenId) == to；tokenURI 与输入一致。
余额：balanceOf 在铸造后 +1，转出后 −1。
转移：非 owner 未授权时 transferFrom 应回滚；approve 后指定地址可代转该 id。
安全转账：接收方为实现了 IERC721Receiver 的合约时应成功；未实现时应失败（教学可部署 OpenZeppelin 的 ERC721Holder 对比）。
