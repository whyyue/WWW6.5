这个页面在链上做了什么？

          

            合约 FairChainLottery 把 ETH 奖池和参与者记在区块链上；随机数由
            Chainlink VRF 在回调 fulfillRandomWords 里产生，再按随机数选中中奖者并发奖。前端不会「算随机数」，只负责读状态、发起
            enter / startLottery / endLottery 交易。
          


          
几个必懂概念（和本页一一对应）

          
    
            
  1. 
              钱包（如 MetaMask）：保存你的私钥，浏览器里通过
              window.ethereum 暴露给网页；点「连接钱包」即请求用户授权账户给本页使用。
            
    2. 
            
  2. 
              Provider：只读通道，可调用 view 函数（如
              entryFee、lotteryState），不花钱。
            
    3. 
            
  3. 
              Signer：能签名「交易」的对象；发 enter、startLottery 等会改链上状态，需付 gas（测试网可用水龙头领测试 ETH）。
            
    4. 
            
  4. 
              合约地址：部署后唯一标识这份合约；必须填对且网络一致（例如 Sepolia 的地址不能拿到主网用）。
            
    5. 
            
  5. 
              ABI：函数名与参数类型的清单；本页用「最小 ABI」数组，只包含用到的函数，ethers 用它来编码调用数据。
            
    6. 
            
  6. 
              payable / msg.value：enter() 要求附带至少
              entryFee 的 ETH；代码里用 { value: fee } 把一笔钱随交易一起发给合约。
            
    7. 
          


          
合约状态机（和按钮关系）

          
    
            
  • CLOSED：未开放或上一轮已结束；owner 可点「开始抽奖」→ OPEN。
    • 
            
  • OPEN：用户可「支付入场费参与」；owner 可「结束并请求随机数」→ CALCULATING。
    • 
            
  • 
              CALCULATING：已向 VRF 请求随机数，等待预言机回调；回调完成后中奖者入账、状态回到
              CLOSED。若长时间不变，检查订阅 LINK/ETH、回调 gas、网络是否拥堵。
            
    • 
          


          
两笔链上交易：请求随机数 vs 真正开奖

          

            点「结束并请求随机数」且 MetaMask 显示成功，只表示你的合约已成功调用
            requestRandomWords（链上会更新 latestRequestId）。中奖地址与转奖池发生在第二笔：Chainlink 节点以回调交易调用你合约里的
            fulfillRandomWords，这一步才选出中奖者并发 ETH。若一直处于
            CALCULATING，请到区块浏览器查看是否有针对你合约的失败回调，并检查 VRF 订阅余额、Consumer 是否已添加。
          


          
本地如何预览？

          

            不要用 file:// 直接打开（部分环境限制钱包注入）。在项目目录执行：
          

          
npx --yes serve .

          
浏览器打开提示的本地地址，再连接钱包即可。
