Day16 Game Plugin System (PluginStore + 多插件示例)

这个页面用最简单的原生 HTML + JS，演示：
PluginStore 作为“玩家档案 + 插件调度中心”，以及 Achievements / Weapon / Friends / BattleLog / TokenInventory 五类插件的交互方式。

1. 连接钱包 / 配置 PluginStore & 插件地址

部署完合约后，把地址填到下面。
注意：只有 PluginStore 的 owner 才能调用 registerPlugin 注册或替换插件。

未连接

2. 玩家档案 Profile（由 PluginStore 直接管理）

这里直接调用 PluginStore 的 setProfile / getProfile。
name / avatar 字段只是字符串，可以自己约定头像 URL 或者 Emoji 等。

3. 成就插件 AchievementsPlugin

通过 PluginStore.runPlugin / runPluginView 调用插件：
- 写：setAchievement(address,string)
- 读：getAchievement(address)

4. 武器插件 WeaponStorePlugin

同样通过 PluginStore 调用：
- 写：setWeapon(address,string)
- 读：getWeapon(address)

5. 好友系统 FriendsPlugin

这里把“好友标识”设计成 string，可以填写地址或昵称。
调用插件函数：addFriend(address,string) / removeFriend(address,string) / checkFriend(address,string)。

6. 战斗日志 BattleLogPlugin

为了简化，我们让前端把一整条战斗描述拼成一个 summary 字符串传入。
- 写：addBattleLog(address,string) （summary）
- 读：getLastBattleLog(address)

7. 代币库存 TokenInventoryPlugin (ERC-20)

这个插件本身在构造函数中固定了某个 ERC-20 token 地址。
这里直接与 TokenInventoryPlugin 合约交互（不通过 PluginStore 代理），演示“插件再去调用 Token 合约”的模式。
