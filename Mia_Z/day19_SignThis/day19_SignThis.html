Day19 SignThis 教学面板

与 day19_SignThis.sol 配套：messageHash = keccak256(abi.encodePacked(attendee, address(this), eventName))， 钱包按 personal_sign 规则对 32 字节摘要加前缀后签名；链上用 ecrecover 校验签名者是否为 organizer。

技术栈：原生 HTML + JS，ethers v6（CDN）。亦可使用仓库内 day19_signScript.mjs 在 Node 侧生成 v/r/s。

只读调用可以怎么连节点？
方式一：填 RPC URL（任意 JSON-RPC 端点，也可用下方「公共示例」免注册）。
方式二：RPC 留空，先点「连接 MetaMask」，只读会走钱包当前网络自带的节点，无需 Infura/Alchemy 账号。
调用 view 函数时不广播交易、不花 gas；签名与写入仍必须用钱包。

2. 步骤教学（按顺序做）

阶段 A — 只读

  1. 在 Remix / Hardhat 等部署 SignThis；部署账户即为链上 organizer。
  2. 填入 链 ID 与 合约地址、attendee。只读二选一：填公共 RPC（见第 1 节示例），或先连接 MetaMask 后把 RPC 留空。
  3. 点「读取链上数据」：应看到 eventName、organizer、messageHash，且「本地复算」与链上一致。

阶段 B — 链下签名（必须 organizer 的私钥；浏览器里 = 连 MetaMask 用该地址签）

  1. 点「连接 MetaMask」，切换到同一链；连接地址应为 organizer（页面会提示是否一致）。
  2. 点「对 messageHash 签名」，在钱包里确认；得到 v, r, s。（也可用 day19_signScript.mjs + 组织者私钥离线签出同样结果。）

阶段 C — 链上校验与写入

  1. 点「verifySignature（只读）」：通过当前只读通道（自定义 RPC 或已连钱包）预检签名；若填了 RPC 可不连钱包做此步。
  2. 点「checkInWithSignature」：需已连接钱包，由任意账户付 gas；交易成功后 attendee 被记为已签到（合约不检查 msg.sender 是否为 attendee）。

4. 组织者签名

须先成功读取 messageHash。当前连接地址应与链上 organizer 一致。

5. 链上写入 / 预检

verifySignature 为只读 eth_call，不发起交易；若填写了自定义 RPC 则不会为读数据弹钱包。checkInWithSignature 使用已连接钱包付 gas。
