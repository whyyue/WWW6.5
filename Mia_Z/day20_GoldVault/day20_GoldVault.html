Day20 GoldVault 重入教学面板

与 day20_GoldVault.sol、day20_GoldThief.sol 配套：重入（Reentrancy）指外部合约在收到 ETH 时通过 receive/fallback 再次调用你的函数。call 会把执行流交给对方，类似嵌入式里「回调 / 中断里又触发了同一条业务路径」。

CEI（Checks-Effects-Interactions）：先检查、再改合约状态（如把 goldBalance 清零）、最后再 call 转币。 vulnerableWithdraw 先转账再清零，映射里余额仍 > 0，重入可再次提款。safeWithdraw 先清零再转账；重入时金额为 0，无法重复掏空。合约里还有 nonReentrant 互斥锁，可作纵深防御。

call 转发全部 gas，对方逻辑可任意执行；transfer/send 仅 2300 gas，仅够记一笔日志，教学里常用来降低重入面，但现代合约更推荐 CEI + 重入锁。

演示要点：若金库里只有盗贼刚存入的「攻击押金」那一笔，第一次转出后合约可能已空，重入会失败。要看清多轮重入，需先用另一账户向 GoldVault.deposit 注入明显多于单笔攻击押金的 ETH，再让盗贼执行 attackVulnerable。链上门槛为 GoldThief.MIN_ATTACK_WEI（当前与前端一致为 0.01 ETH）。attackCount < 5 为教学上限。

技术栈：原生 HTML + JS，ethers v6（CDN）。若浏览器直接打开文件时 RPC 报 CORS，可用 VS Code Live Server 或 npx serve 在本地起静态服务。

2. 步骤教学（按顺序做）

阶段 A — 部署

  1. 在 Remix / Hardhat 部署 GoldVault，再部署 GoldThief，构造参数填入 Vault 地址。
  2. 将 Vault、Thief 地址填入本节上方；链 ID 与 RPC 按实际网络填写。

阶段 B — 注资（脆弱演示必做）

  1. 用账户 A连接钱包，向 Vault deposit 一定量 ETH（建议明显多于 0.01 ETH，例如 ≥ 0.05 ETH，便于观察多轮重入）。
  2. 点「读取链上数据」确认 goldBalance 与合约 ETH 余额。

阶段 C — 脆弱提款与攻击

  1. 用盗贼 owner账户连接（即部署 GoldThief 的地址）。
  2. 在盗贼卡片发送 attackVulnerable，附带的 ETH ≥ 0.01（默认 0.01 ETH，须 ≥ 链上 MIN_ATTACK_WEI）。观察 attackCount、盗贼余额、Vault 余额变化。
  3. 需要时用 stealLoot 把盗贼合约内 ETH 取回 owner。

阶段 D — 对比安全路径

  1. 可再次注资后，对同一盗贼调用 attackSafe（走 safeWithdraw）。结合源码说明：先清零再 call，重入无法重复提款；nonReentrant 为额外保险。
  2. 也可用普通账户自行尝试 vulnerableWithdraw / safeWithdraw 对比行为（须有 goldBalance）。

6. 测试验收清单（自学对照）

部署：Vault 与 Thief 部署成功，Thief 的 targetVault 指向 Vault。
注资：账户 A 向 Vault deposit 足额 ETH 后，再执行 attackVulnerable。
预期：attackCount ≥ 2；盗贼合约 ETH 增加；Vault 总损失超过盗贼「公平份额」时可定性为重入放大。
stealLoot：owner 取回盗贼合约余额。
attackSafe：不应出现与脆弱路径相同的多轮掏空；结合「先 Effects 再 Interactions」讲解。
